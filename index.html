What is Bread.CSS?

Bread.CSS is a follow up CSS framework to our in-class lab creating Butter.CSS to learn about SaSS partials and framework organization.

It was made by Taro Routly, Taylor Baker, Lauren Brunet, and Paul Decker for MTM6407 at Algonquin College in the Interactive Media Design program.

Installation

Direct CSS Implementation

If you are looking to directly import this theme without changing variables, we recommend downloading the already compiled CSS file.

There are two options for CSS implementation, a readable version with proper formatting, or a smaller minified CSS file. Both are available for download at the links below.

These files should be added to your project by adding the file into your project's CSS folder, then add the link to the stylesheet in the head of your document using the relative path of the bread.css file.

Example: <link rel="stylesheet" href="./css/bread.css">

Download readable CSS here Download minified CSS here

Source Code Implementation

If you are looking to modify the theme to fit your project's needs, we recommend that you download our source code .zip file, which can be downloaded at the button below

This option may be difficult for people new to SaSS, so we recommend only people with an understanding of complex CSS and beginner level SaSS knowledge do this path. Below has more information on the variables.

SaSS MUST be installed on your machine to modify the source code and see updates in your compiled CSS.

Adding Fonts

To add new fonts to the theme, go to to the src/fonts folder and add a new file. The naming convention for this file should be _font-name.scss, replacing 'font-name' with the actual name of your font, in lowercase. Then, add the new file name to src/fonts/index.scss using @forward './font-name';, without the _ or .scss extention.

To overwrite the current default for body and headings, go to src/variables/_typography.scss and change the values of the $fonts map to the new font name as you would declare it in the CSS property `font-family`, without the sans-serif or serif declaration.

Adding Icons

To add a new icon set without getting rid of the old one, go to to the src/icons/ folder and add a new file. The naming convention for this file should be _icon-kit-name.scss, replacing 'icon-kit-name' with the actual name of your icon kit, in lowercase. Then, add the new file name to src/icons/index.scss as @forward './icon-kit-name;'

To overwrite the current icon set with a new one, change the @import statement in src/icons/_icons.scss to the one used for the new icon set.

Modifying Colors

To modify the theme colors, go to src/variables/_colors.scss and modify both the $theme-colors and $theme-colors-hovers map with your new colours and an associated color for its hover state.

Components may need to be updated to adjust for colour contrast issues.

Components Showcase

This section will show examples of the different components in our framework.

Buttons

To use the Buttons components add the class of btn to a <button> tag or an <a> tag for the default styling.

There are 9 color variants of the button component, which can be used by adding the theme color name to the end of the btn class.

Example: <button class="btn-secondary">

Card

To use the Card component in your project first add the card class to the container (usually a div) you want to become the card.

If you plan to include an image in your card, add the img tag as a child of the card container.

Within the card container, create another div with the class of card-body. Put the remaining tags inside this container.

Add the class of card-title to the tag you want to be the title of the card. Add the class of card-content to the tag you want to contain the main content of the card. Add other tags as necessary.

You can modify the styling of the card component (such as color, spacing, etc) using utility classes.

Infobox

To use the infobox component, add the class infobox to a div container.

Within that container, add two div tags, with the tags infobox-header and infobox-body respectively. In div.infobox-header, add the topic of the infobox, with the option to include an icon of your choice on the right side of the header. In div.infobox-body, add the content of the infobox.

There are 9 color variants of the infobox component, which can be used by adding the theme color name to the end of the infobox class.

Example: <div class="infobox-secondary"><div/>

Breadcrumbs

To use the breadcrumbs component, add the class breadcrumbs to a div container.

Within that container, add an ul tag with a class of breadcrumbs-list. Within that list, add each link in your breadcrumbs as an anchor tag inside a list item with the class breadcrumbs-item.

Example: <li class="breadcrumbs-item"><a href="#">1st Path</a></li>

Add the class current-item to the list item that represents the current page.

There are 9 color variants of the breadcrumbs component, which can be used by adding the theme color name to the end of the breadcrumbs class.

Example: <div class="breadcrumbs-secondary"></div>
